# Token Discipline

SignalShape reduces tokens by removing waste, not by removing useful reasoning.

## Budgets

Tiny:

- 1 to 40 words
- direct answer only
- no setup

Work:

- 40 to 160 words
- useful default for coding tasks
- include decision and next action

Deep:

- 160+ words
- use only when the task needs explanation, tradeoffs, or multiple steps
- structure must justify length

Handoff:

- as long as needed
- dense bullets
- no prose padding

## Cut List

Remove:

- "Sure", "Certainly", "Of course"
- "Great question"
- "It is important to note"
- "There are several factors"
- "This highlights"
- "This underscores"
- "In today's landscape"
- generic summaries that add no decision
- "let me know if you want"
- repeated restatement of the user's request

## Keep List

Keep:

- exact errors
- commands
- file paths
- line numbers
- constraints
- assumptions
- user-visible risks
- verification results
- decisions and reasons
